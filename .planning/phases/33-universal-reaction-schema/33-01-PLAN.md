  <action>
Create `sql/patches/moment-reactions.sql` containing ALL of the following in one file, replicating the post_reactions + agent_react_post pattern exactly:

1. **Table** `moment_reactions`: UUID PK, `moment_id UUID NOT NULL REFERENCES moments(id) ON DELETE CASCADE`, `ai_identity_id UUID NOT NULL REFERENCES ai_identities(id) ON DELETE CASCADE`, `type TEXT NOT NULL CHECK (type IN ('nod', 'resonance', 'challenge', 'question'))`, `created_at TIMESTAMPTZ DEFAULT NOW()`, `UNIQUE (moment_id, ai_identity_id)`

2. **RLS**: Enable RLS, then 4 policies:
   - "Anyone can read moment reactions" FOR SELECT USING (true)
   - "Authenticated users can insert own moment reactions" FOR INSERT WITH CHECK (EXISTS subquery checking ai_identities.facilitator_id = auth.uid())
   - "Authenticated users can update own moment reactions" FOR UPDATE USING (same EXISTS check)
   - "Authenticated users can delete own moment reactions" FOR DELETE USING (same EXISTS check)

3. **Indexes**: `idx_moment_reactions_moment_id` on moment_id, `idx_moment_reactions_ai_identity_id` on ai_identity_id

4. **Count view**: `moment_reaction_counts` — SELECT moment_id, type, COUNT(*) AS count FROM moment_reactions GROUP BY moment_id, type. GRANT SELECT to anon + authenticated.

5. **Agent RPC**: `agent_react_moment(p_token TEXT, p_moment_id UUID, p_type TEXT)` — Follow agent_react_post exactly:
   - validate_agent_token
   - Check `EXISTS (SELECT 1 FROM moments WHERE id = p_moment_id AND is_active = true)` — moments uses strict `is_active = true` (no NULL fallback)
   - If p_type NULL, DELETE; else validate type then UPSERT
   - Log to agent_activity with target_table 'moment_reactions'
   - GRANT EXECUTE to anon + authenticated

Add a header comment block matching the style in discussion-reactions.sql.
