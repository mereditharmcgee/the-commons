**Part A: Add onboarding banner HTML to dashboard.html**

Add a `<section id="onboarding-banner">` as the first child of the main dashboard content area (before the profile/identity sections). Structure per 38-RESEARCH.md Pattern 1:
- Header with "Welcome to The Commons" title and a [Dismiss] button (btn btn--ghost btn--small)
- Container div `id="onboarding-steps"` for JS-rendered step items
- Initially hidden via `style="display:none;"` (JS reveals it)
- Add CSS for the banner in a `<style>` block in the page OR in css/style.css. IMPORTANT: Check if dashboard.html has a CSP meta tag with script hashes. If it does, do NOT add inline style tags — add CSS to css/style.css instead. Add classes: `.onboarding-banner` (border, padding, background matching dark theme), `.onboarding-step` (flex row), `.onboarding-step--done` (checkmark styling with accent-gold color)

**Part B: Add banner logic to dashboard.js**

Create `renderOnboardingBanner(identities, hasToken, hasActivity)` function:
1. Check `localStorage.getItem('tc_onboarding_dismissed')` — if set, return immediately
2. Evaluate 3 steps:
   - "Create an identity": `identities.length > 0`
   - "Generate an agent token": `localStorage.getItem('tc_onboarding_token_generated') === '1'`
   - "Bring your first AI": check if any identity has stats with `post_count > 0` (use data already available from ai_identity_stats, or check identities array)
3. If all 3 complete, auto-dismiss: `localStorage.setItem('tc_onboarding_dismissed', '1'); return;`
4. Render steps with checkmarks for completed ones, numbered for incomplete
5. Step 1 links to `#identities` anchor, Step 2 links to `#tokens` anchor, Step 3 links to `agent-guide.html`
6. Wire dismiss button to set localStorage and hide banner

Call `renderOnboardingBanner()` after `loadIdentities()` completes — pass identities array. For hasToken, check localStorage. For hasActivity, query ai_identity_stats for any identity with post_count > 0.

Also: When tokens are generated in `loadTokens()`, set `localStorage.setItem('tc_onboarding_token_generated', '1')` so the banner can detect it.

**Part C: Fix dashboard.js state handling (ONBD-04, ONBD-05)**

Search dashboard.js for all occurrences of inline state HTML patterns:
- `innerHTML = '<p class="text-muted">Loading...</p>'` or similar → replace with `Utils.showLoading(container)`
- Inline error messages → replace with `Utils.showError(container, message, { onRetry: retryFn })`
- Empty state messages → replace with `Utils.showEmpty(container, title, subtitle)` with CTA where appropriate

Per 38-RESEARCH.md, dashboard.js has 8+ inline loading patterns. Replace each one. Preserve the existing behavior — just use the shared helpers for visual consistency.

IMPORTANT: Do NOT add inline `<script>` tags to dashboard.html if it has a CSP meta tag. All JS goes in dashboard.js.
