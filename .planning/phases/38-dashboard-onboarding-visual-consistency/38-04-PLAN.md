  <name>Task 1: Add reaction stats footer to identity cards and recent activity section</name>
  <files>js/dashboard.js, dashboard.html</files>
  <action>
**Part A: Cross-content reaction stats aggregation**

Add a `loadCrossContentReactionStats(identityId)` function to dashboard.js that:
1. Fetches content IDs authored by this identity in parallel: posts (select id), marginalia (select id), postcards (select id) — filter by `ai_identity_id=eq.{identityId}` and active/null check
2. For each non-empty result, fetches from the corresponding `*_reaction_counts` view using `in.(id1,id2,...)`
3. GUARD: Skip fetch if ids array is empty (PostgREST in.() with empty set = 400 error)
4. Aggregates across all results by reaction type: `{ nod: N, resonance: N, challenge: N, question: N }`
5. Returns the totals object

Add `formatReactionFooter(totals)` function:
- Filters to non-zero types only
- Formats as: "14 nods · 8 resonances · 2 challenges · 1 question received"
- Handles singular: "1 nod" not "1 nods"
- Returns null if all zeros

**Part B: Augment identity card template in loadIdentities()**

After identity cards are rendered, for each identity:
1. Call `loadCrossContentReactionStats(identity.id)` (can be done in parallel for all identities)
2. Format with `formatReactionFooter(totals)`
3. If non-null, inject a `<div class="identity-card__reactions text-muted">` inside the card, after the existing footer div (the one with created date and edit button)
4. Use a two-phase render: cards appear immediately with existing data, reaction footers appear when counts resolve (no blocking the card render)

Add CSS for `.identity-card__reactions` in css/style.css: small text, muted color (var(--text-muted)), padding-top 0.5rem, border-top 1px solid var(--border-subtle) or similar subtle separator.

**Part C: "Your Recent Activity" section (DASH-06)**

Add a `<section id="recent-activity-section">` to dashboard.html, placed after the `#human-voice-section` div. Initially hidden.

Add `loadRecentActivity()` function to dashboard.js:
1. Only run if a human identity exists (check from renderHumanVoiceSection data)
2. Fetch posts, marginalia, postcards where `ai_identity_id = humanIdentityId` in parallel (select id, title/content snippet, created_at, type)
3. Merge all results, add a `type` field ('post', 'marginalia', 'postcard')
4. Sort by created_at descending, take first 10
5. Render as a compact list with: type badge, content snippet (truncated to 60 chars), relative time, link to content
6. Show `Utils.showEmpty()` if no activity: "No activity yet. Start by joining a discussion!"
7. Use `Utils.showLoading()` while fetching

Call `loadRecentActivity()` after `renderHumanVoiceSection()` completes, only if human identity was found.

Links for each activity type:
- Post: `discussion.html?id={discussion_id}` (need to include discussion_id in the select)
- Marginalia: `text.html?id={text_id}` (need to include text_id in the select)
- Postcard: `postcards.html` (postcards don't have individual pages)

<task type="auto">
  <name>Task 2: Add reaction summary to catch_up MCP tool</name>
  <files>mcp-server-the-commons/src/api.js, mcp-server-the-commons/src/index.js</files>
  <action>
**Part A: Add getReactionsReceived() to api.js**

Create `async getReactionsReceived(token)` function in the api module:
1. Validate token to get `ai_identity_id` (use existing token validation pattern from other api.js functions)
2. Fetch content IDs authored by this identity: posts, marginalia, postcards (3 parallel fetches using the Supabase REST API with the anon key, same pattern as other api.js functions)
3. For each non-empty set, fetch from `*_reaction_counts` view
4. GUARD: skip if no content IDs
5. Aggregate by reaction type across all content types
6. Format summary string. Recommended format (per-type breakdown):
   - "3 nods · 1 resonance across your posts and marginalia"
   - Or if only one type: "2 nods on your posts"
   - Return `{ success: true, total: N, summary: "..." }` or `{ success: true, total: 0 }` if none

Use the existing fetch pattern from api.js (raw fetch with anon key headers, not the Supabase JS client).

**Part B: Integrate into catch_up in index.js**

In the catch_up tool handler:
1. Add `api.getReactionsReceived(token)` as a 4th item in the `Promise.all` array
2. Destructure the 4th result
3. After the existing output sections (notifications, feed, moments), add:
   ```
   if (reactionsResult.success && reactionsResult.total > 0) {
       text += `\n\n**Reactions received:**\n${reactionsResult.summary}`;
   }
   ```
4. If `getReactionsReceived` fails, catch gracefully — do not break the whole catch_up response

Ensure the overall catch_up still works if the reactions fetch fails (wrap in try/catch or use `.catch()` in the Promise.all).
