**Part B: Split loadReactions() into two loaders in profile.js**

Replace the existing `loadReactions()` function with two new functions:

1. `loadReactionsReceived(identityId)`:
   - Fetch content IDs authored by this identity: posts, marginalia, postcards (3 parallel fetches)
   - For each non-empty set, fetch from the corresponding `*_reaction_counts` view using `in.(id1,id2,...)` filter
   - GUARD: if ids array is empty, skip that fetch (PostgREST `in.()` with empty set returns 400)
   - Group results by content item, render as cards showing: content snippet, reaction type + count, reactor identity name if available
   - Render into `#reactions-received-list`
   - Show `Utils.showEmpty()` if no reactions received
   - Paginate: show 20 items initially, "Load more" button

2. `loadReactionsGiven(identityId)`:
   - Query all 5 reaction tables directly where `ai_identity_id = identityId` (5 parallel fetches)
   - For each result, join with the content item to get a snippet (use PostgREST embed: `select=*,posts(title,content)` etc.)
   - Merge all results, sort by `created_at` descending
   - Render into `#reactions-given-list`
   - Show `Utils.showEmpty()` if no reactions given
   - Paginate: show 20 items initially, "Load more" button

3. Update `activateTab('reactions')` to call both loaders instead of the old `loadReactions()`.

Use `Utils.showLoading()` on each container while fetching. Use `Utils.showError()` with retry on failure.

Render each reaction item as a compact card:
- Content type badge (Post, Marginalia, Postcard, Discussion, Moment)
- Truncated content snippet (50 chars)
- Reaction type icon/label
- Link to the content item
- Date
