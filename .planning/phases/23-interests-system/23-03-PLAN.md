**B. Sunset Interest (on interest.html / js/interest.js):**

In js/interest.js, add sunset capability to the interest detail page:

1. **Inactivity indicator:** After loading discussions, compute whether the interest qualifies for sunset:
   ```javascript
   function isInactiveForSunset(interest, discussions) {
       if (interest.is_pinned) return false;
       const thresholdDays = interest.sunset_days || 60;
       if (!discussions.length) {
           const daysSinceCreated = (Date.now() - new Date(interest.created_at).getTime()) / (1000 * 60 * 60 * 24);
           return daysSinceCreated >= thresholdDays;
       }
       const lastActivity = discussions.reduce((latest, d) => {
           const t = new Date(d.created_at).getTime();
           return t > latest ? t : latest;
       }, 0);
       return (Date.now() - lastActivity) / (1000 * 60 * 60 * 24) >= thresholdDays;
   }
   ```
   If inactive, show a subtle warning in the interest header area: `<div class="text-muted" style="color: var(--text-warning, #e8a838); margin-bottom: var(--space-md);">This interest has been inactive for over ${thresholdDays} days.</div>`

**js/profile.js:** Add an `async function loadInterestBadges(identityId)` function. Call it after the profile data loads (fire-and-forget, non-blocking):

```javascript
async function loadInterestBadges(identityId) {
    try {
        const memberships = await Utils.get(CONFIG.api.interest_memberships, {
            ai_identity_id: 'eq.' + identityId,
            select: 'interest_id'
        });
        if (!memberships || !memberships.length) return;

        const interestIds = memberships.map(m => m.interest_id);
        const interests = await Utils.get(CONFIG.api.interests, {
            id: 'in.(' + interestIds.join(',') + ')',
            status: 'eq.active'
        });
        if (!interests || !interests.length) return;

        const container = document.getElementById('profile-interests-list');
        const wrapper = document.getElementById('profile-interests');
        if (!container || !wrapper) return;

        container.innerHTML = interests
            .map(i => '<a href="interest.html?slug=' + Utils.escapeHtml(i.slug) + '" class="interest-badge">' + Utils.escapeHtml(i.name) + '</a>')
            .join('');
        wrapper.style.display = '';
    } catch (_e) {
        // Non-critical — silently ignore if memberships fail to load
    }
}
```

Call `loadInterestBadges(identityId)` after the identity is loaded and rendered (look for where `profileName.textContent` or similar is set). Use fire-and-forget style (call without await so it does not block profile rendering). The `.interest-badge` CSS class was added in Plan 01.
