**sql/schema/13-interest-endorsements.sql:** Create the interest_endorsements table for the emerging themes endorsement mechanism:
```sql
-- 13-interest-endorsements.sql
-- Phase 23: Endorsement table for emerging interest themes
CREATE TABLE IF NOT EXISTS interest_endorsements (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    interest_id UUID NOT NULL REFERENCES interests(id) ON DELETE CASCADE,
    facilitator_id UUID NOT NULL REFERENCES facilitators(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(interest_id, facilitator_id)
);
ALTER TABLE interest_endorsements ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Anyone can read endorsements" ON interest_endorsements FOR SELECT USING (true);
CREATE POLICY "Facilitators can endorse emerging interests" ON interest_endorsements
    FOR INSERT WITH CHECK (auth.uid() IS NOT NULL AND facilitator_id = auth.uid());
CREATE POLICY "Facilitators can remove their endorsement" ON interest_endorsements
    FOR DELETE USING (facilitator_id = auth.uid());
CREATE INDEX IF NOT EXISTS idx_interest_endorsements_interest ON interest_endorsements(interest_id);
```

**js/interests.js:** Create new file as an async IIFE (matching discussions.js pattern). Implements the full interests browse page:

1. **DOM references:** Get `#interests-grid` container and `#emerging-section` container from interests.html.

2. **Data fetching:** Fetch three datasets in parallel with `Promise.all`:
   - `Utils.get(CONFIG.api.interests, { order: 'name.asc' })` — all interests (filter client-side by status)
   - `Utils.get(CONFIG.api.interest_memberships, { select: 'interest_id' })` — all memberships for counts
   - `Utils.get(CONFIG.api.discussions, { select: 'id,interest_id,created_at', is_active: 'eq.true' })` — all active discussions for counts and recency

3. **Compute aggregates:**
   - `memberCounts` object: count memberships per interest_id
   - `discussionCounts` object: count discussions per interest_id (include `interest_id IS NULL` discussions in General's count)
   - `lastActivity` object: most recent `created_at` per interest_id (for recency indicator)

4. **Render active interests grid:** Filter interests where `status === 'active'`. Sort by `is_pinned` first (pinned at top), then by name. Map to card HTML:
   ```html
   <a href="interest.html?slug=${Utils.escapeHtml(interest.slug)}" class="interest-card">
       <h3 class="interest-card__name">${Utils.escapeHtml(interest.name)}</h3>
       ${interest.description ? `<p class="interest-card__description">${Utils.escapeHtml(interest.description)}</p>` : ''}
       <div class="interest-card__meta">
           <span>${memberCount} ${memberCount === 1 ? 'member' : 'members'}</span>
           <span>${discussionCount} ${discussionCount === 1 ? 'discussion' : 'discussions'}</span>
           ${lastActivityStr ? `<span>${lastActivityStr}</span>` : ''}
       </div>
   </a>
   ```
   For the last activity indicator, use `Utils.formatDate(lastActivity, true)` for the short date. If no discussions exist for that interest, omit the activity span.

   If no active interests exist, use `Utils.showEmpty(container, 'No interests yet', 'Communities will appear here as they are created.')`.

5. **Render emerging themes section:** Filter interests where `status === 'emerging'`. If none, hide the `#emerging-section` container entirely (`style.display = 'none'`). If any exist, show the section with heading "What's Brewing" and render emerging cards. Each emerging card includes:
   - Interest name and description (same as active cards)
   - An "Endorse" button (class `emerging-card__endorse`) showing the endorsement count
   - If the user is logged in and has already endorsed, the button gets class `endorsed` and text changes to "Endorsed (N)"

6. **Endorsement logic (auth-gated):**
   - After auth resolves, fetch endorsements for the current user: `Auth.getClient().from('interest_endorsements').select('interest_id').eq('facilitator_id', Auth.getUser().id)`
   - Fetch all endorsement counts: `Utils.get(CONFIG.api.interest_endorsements, { select: 'interest_id' })` and count per interest_id
   - Wire click handlers on endorse buttons: if not endorsed, INSERT via `Auth.getClient().from('interest_endorsements').insert(...)` catching error code 23505 for duplicates; if already endorsed, DELETE. Toggle button class and update count text immediately (optimistic UI).
   - If user is NOT logged in, endorse buttons should still show with counts but clicking shows an alert: "Log in to endorse emerging interests."

7. **Auth integration:** Call `Auth.init()` without await (public page). Set up a listener for when auth resolves to wire endorsement logic.

8. **Error handling:** Wrap the main fetch in try/catch. On error, use `Utils.showError(container, 'Could not load interests.', { onRetry: () => location.reload() })`.
