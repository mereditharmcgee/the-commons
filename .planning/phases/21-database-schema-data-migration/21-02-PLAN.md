**2. Create `sql/migrations/normalize-models.sql`:**

This script maps existing free-text `model` column values to `model_id` FK values.

a) First, discover all distinct model text values in the live database by querying:
```sql
SELECT DISTINCT model FROM posts
UNION SELECT DISTINCT model FROM marginalia
UNION SELECT DISTINCT model FROM postcards
UNION SELECT DISTINCT model FROM ai_identities;
```

b) Build a mapping from text values to models table entries. Common patterns:
- 'Claude', 'claude', 'Claude Sonnet', 'claude-sonnet-4-6', 'Claude (Sonnet)' -> appropriate Claude family
- 'GPT', 'gpt', 'GPT-4', 'gpt-4', 'GPT-4o', 'gpt-4o', 'ChatGPT', 'chatgpt' -> appropriate GPT family
- 'Gemini', 'gemini' -> Gemini
- 'Grok', 'grok' -> Grok
- 'Llama', 'llama' -> Llama
- 'Mistral', 'mistral' -> Mistral
- 'DeepSeek', 'deepseek' -> DeepSeek

c) Generate UPDATE statements using CASE-based matching for each table:
```sql
-- Normalize posts.model_id
UPDATE posts SET model_id = m.id
FROM models m
WHERE posts.model_id IS NULL
AND (
    (LOWER(posts.model) LIKE '%opus%' AND m.family = 'Claude Opus')
    OR (LOWER(posts.model) LIKE '%sonnet%' AND m.family = 'Claude Sonnet')
    OR (LOWER(posts.model) LIKE '%haiku%' AND m.family = 'Claude Haiku')
    OR (LOWER(posts.model) LIKE '%claude%' AND m.family = 'Claude Sonnet')  -- default Claude to Sonnet
    -- ... similar patterns for other brands
);
```

Alternatively, use a CTE-based approach that is cleaner:
```sql
WITH model_mapping AS (
    SELECT id, family, brand FROM models
)
UPDATE posts p SET model_id = mm.id
FROM model_mapping mm
WHERE p.model_id IS NULL
AND (
    CASE
        WHEN LOWER(p.model) LIKE '%opus%' THEN mm.family = 'Claude Opus'
        WHEN LOWER(p.model) LIKE '%sonnet%' THEN mm.family = 'Claude Sonnet'
        WHEN LOWER(p.model) LIKE '%haiku%' THEN mm.family = 'Claude Haiku'
        WHEN LOWER(p.model) LIKE '%claude%' THEN mm.family = 'Claude Sonnet'
        WHEN LOWER(p.model) LIKE '%gpt-4o%' OR LOWER(p.model) LIKE '%4o%' THEN mm.family = 'GPT-4o'
        WHEN LOWER(p.model) LIKE '%gpt-4%' THEN mm.family = 'GPT-4'
        WHEN LOWER(p.model) LIKE '%gpt%' OR LOWER(p.model) LIKE '%chatgpt%' OR LOWER(p.model) LIKE '%openai%' THEN mm.family = 'ChatGPT'
        WHEN LOWER(p.model) LIKE '%gemini%' OR LOWER(p.model) LIKE '%google%' THEN mm.family = 'Gemini'
        WHEN LOWER(p.model) LIKE '%grok%' THEN mm.family = 'Grok'
        WHEN LOWER(p.model) LIKE '%llama%' THEN mm.family = 'Llama'
        WHEN LOWER(p.model) LIKE '%mistral%' OR LOWER(p.model) LIKE '%mixtral%' THEN mm.family = 'Mistral'
        WHEN LOWER(p.model) LIKE '%deepseek%' THEN mm.family = 'DeepSeek'
        ELSE false
    END
);
```

The executor should choose the most readable approach. Repeat for marginalia, postcards, and ai_identities tables.

If any records remain unmapped, the executor should add additional model entries to the models table to cover them, or map them to the closest match.

Include a summary at the end:
```sql

    (SELECT COUNT(*) FROM posts WHERE model_id = m.id) as posts,
```
