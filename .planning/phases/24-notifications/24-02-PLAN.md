  <action>
**Create js/notifications.js** as a self-contained module (IIFE pattern like nav.js) that manages the notification dropdown popover. This file will be included via script tag on every page (after auth.js, before page-specific scripts).

The module should:

1. **Initialize after auth resolves.** Listen for the `authStateChanged` custom event (dispatched by Auth.updateUI). When the user is logged in, set up the dropdown. When logged out, tear it down.

2. **Create the dropdown DOM.** On first init, create a `div.notification-dropdown` element and append it as a sibling right after the `#notification-bell` button (inside `.user-menu`). The dropdown should contain:
   - A header with "Notifications" title and a "Mark all read" button
   - A list container for notification items (`.notification-dropdown__list`)
   - A footer with a "See all" link pointing to `dashboard.html#notifications`
   - An empty state message ("No unread notifications") shown when the list is empty

3. **Toggle on bell click.** Add a click handler to `#notification-bell` that:
   - If dropdown is closed: fetch unread notifications (Auth.getNotifications with limit=10, unreadOnly=true), render them into the list, then show the dropdown by adding class `notification-dropdown--open`
   - If dropdown is open: close it by removing the class
   - Use `e.stopPropagation()` to prevent the document click handler from immediately closing it

4. **Close on outside click.** Add a document click listener that closes the dropdown if the click target is not inside the dropdown or the bell button. This is similar to the hamburger menu pattern in nav.js.

5. **Render notification items.** Each notification item should show:
   - Title (escaped via Utils.escapeHtml)
   - Relative time (use a simple helper: "just now", "Xm ago", "Xh ago", "yesterday", or date string)
   - The entire item is wrapped in an anchor tag linking to the notification's `link` field (if it exists and passes a safe URL check -- starts with a known page name like "discussion.html", "profile.html", etc., or is a relative path)
   - Clicking the item: call Auth.markAsRead(id), decrement the badge count client-side, then navigate

6. **Mark all read.** The "Mark all read" button calls Auth.markAllAsRead(), clears the dropdown list, shows empty state, and sets badge to 0 (hide badge).

7. **Relative time helper.** Include a small function that converts ISO timestamps to relative strings:
   - < 1 minute: "just now"
   - < 60 minutes: "Xm ago"
   - < 24 hours: "Xh ago"
   - < 48 hours: "yesterday"
   - Otherwise: formatted date (e.g., "Mar 4")

8. **Safe URL validation.** Only allow links that are relative paths (no http:// or javascript:). Check that the link starts with a known page name or is a simple relative URL.

**Update css/style.css** -- Add a new section "NOTIFICATION DROPDOWN" after the existing notification-bell styles (around line 3175). Add these classes:

- `.notification-dropdown`: Hidden by default (`display: none`), positioned absolutely below the bell icon. Use `position: absolute; top: 100%; right: 0; width: 360px; max-height: 480px; overflow-y: auto; background: var(--bg-primary); border: 1px solid var(--border-medium); border-radius: 8px; box-shadow: 0 8px 24px rgba(0,0,0,0.4); z-index: 1000;`
- `.notification-dropdown--open`: `display: block`
- `.notification-dropdown__header`: Flex row with space-between, padding, border-bottom. Title in var(--font-serif).
- `.notification-dropdown__mark-all`: Ghost button style, small font, color var(--text-muted), hover var(--accent-gold).
- `.notification-dropdown__list`: List container, no list-style.
- `.notification-dropdown__item`: Styled as a block anchor, padding, border-bottom, hover background var(--bg-elevated). Unread items get left border accent (3px solid var(--accent-gold)).
- `.notification-dropdown__item--unread`: Background var(--accent-gold-glow), left border 3px solid var(--accent-gold).
- `.notification-dropdown__title`: Font-weight 500, color var(--text-primary), font-size 0.875rem.
- `.notification-dropdown__time`: Font-size 0.75rem, color var(--text-muted).
- `.notification-dropdown__empty`: Centered text, padding, color var(--text-muted), font-style italic.
- `.notification-dropdown__footer`: Text-align center, padding, border-top. "See all" link in var(--accent-gold).
- Mobile responsive: At max-width 600px, dropdown should be `width: calc(100vw - 2rem); right: -1rem;` so it doesn't overflow on small screens.
- The `.notification-bell` parent needs `position: relative` added (it may already have it -- check and add if not).

**Update js/nav.js** -- The notification dropdown needs the bell's parent to have `position: relative` for absolute positioning. The bell button is inside `.user-menu` which is inside `.site-nav__auth`. Add `position: relative` to the `.notification-bell` CSS (it already has `position: relative` from line 3145 -- confirmed). No nav.js changes needed for positioning.

However, nav.js should close the notification dropdown when the hamburger menu opens (and vice versa). Add a small integration: when the hamburger opens, dispatch or check for the dropdown and close it. The simplest approach: notifications.js listens for hamburger click and closes dropdown.

Actually, keep nav.js minimal -- do NOT modify nav.js. The notifications.js module handles its own open/close state. It will close the dropdown on any outside click, which naturally handles hamburger interaction.

**Script inclusion**: notifications.js must be loaded on every page. Since all 30 HTML pages already have the bell icon in their nav HTML, they need the script tag. Rather than modifying 30 HTML files, the best approach per the CONTEXT.md decision is to have nav.js dynamically load notifications.js. But the CONTEXT says "inject bell via nav.js" -- however the bell already exists in the HTML. So the dropdown logic just needs to be available.

**Approach for script loading**: Modify nav.js to dynamically create and append a script tag for `js/notifications.js` at the end of DOMContentLoaded. This way notifications.js is loaded on every page automatically without touching any HTML files. Add this at the end of the nav.js IIFE's DOMContentLoaded handler:

```javascript
// Load notification dropdown module
var notifScript = document.createElement('script');
notifScript.src = 'js/notifications.js';
document.body.appendChild(notifScript);
```

This is lightweight and follows the same external-script pattern nav.js already uses.
