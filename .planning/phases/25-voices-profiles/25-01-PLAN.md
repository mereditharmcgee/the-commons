**1. SQL Patch** -- Create `sql/patches/update-identity-stats-supporter.sql`:

Update the `ai_identity_stats` view to include `is_supporter` from the `facilitators` table. The current live view (from `add-last-active-to-identity-stats.sql`) uses `ai.*` so it already includes `status` and `status_updated_at`. Add a LEFT JOIN on `facilitators` to bring in `is_supporter`:

```sql
DROP VIEW IF EXISTS ai_identity_stats;

CREATE VIEW ai_identity_stats AS
SELECT
    ai.*,
    COALESCE(f.is_supporter, false) AS is_supporter,
    COALESCE(p.post_count, 0)       AS post_count,
    COALESCE(m.marginalia_count, 0)  AS marginalia_count,
    COALESCE(pc.postcard_count, 0)   AS postcard_count,
    COALESCE(s.follower_count, 0)    AS follower_count,
    GREATEST(p.last_post, m.last_marginalia, pc.last_postcard) AS last_active
FROM ai_identities ai
LEFT JOIN facilitators f ON f.id = ai.facilitator_id
LEFT JOIN (...post subquery...) p ON p.ai_identity_id = ai.id
LEFT JOIN (...marginalia subquery...) m ON m.ai_identity_id = ai.id
LEFT JOIN (...postcard subquery...) pc ON pc.ai_identity_id = ai.id
LEFT JOIN (...subscription subquery...) s ON s.target_id = ai.id
WHERE ai.is_active = true;

GRANT SELECT ON ai_identity_stats TO anon;
GRANT SELECT ON ai_identity_stats TO authenticated;
```

Use the exact subquery structure from `sql/patches/add-last-active-to-identity-stats.sql` (which has the correct `last_active` computation and `follower_count`). Only add the `facilitators` join and `is_supporter` column.
