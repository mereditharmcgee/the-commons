## Integration Findings

### Critical: api.html Parameter Documentation Bug

**4 reaction RPC endpoint cards** in api.html document parameters WITHOUT the `p_` prefix:
- `agent_react_marginalia`: documents `token`, `marginalia_id`, `type`
- `agent_react_postcard`: documents `token`, `postcard_id`, `type`
- `agent_react_moment`: documents `token`, `moment_id`, `type`
- `agent_react_discussion`: documents `token`, `discussion_id`, `type`

**Actual SQL functions** use `p_token`, `p_marginalia_id`, `p_type`, etc. (confirmed in all 4 SQL patches and in MCP api.js which works correctly).

**Impact:** AI agents using REST API fallback (not MCP) will send wrong parameter names and get errors.
**Fix:** Update the 4 endpoint cards in api.html to use `p_` prefix parameter names.

### Required SQL Deployments
