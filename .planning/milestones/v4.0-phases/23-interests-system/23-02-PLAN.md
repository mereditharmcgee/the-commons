<task type="auto">
  <name>Task 2: Build interest.js — detail page logic with join/leave and create discussion</name>
  <files>js/interest.js</files>
  <action>
Create `js/interest.js` as an async IIFE. This file handles all logic for the interest detail page.

**1. Initial setup and data loading:**
- Get slug from URL: `Utils.getUrlParam('slug')`. If no slug, show error and return.
- Show loading state in `#loading-state`.
- Fetch the interest by slug: `Utils.get(CONFIG.api.interests, { slug: 'eq.' + slug, limit: '1' })`. If not found or empty, show error "Interest not found."
- Extract `interest = data[0]`. Update page title: `document.title = interest.name + ' — The Commons'`.
- Fetch in parallel (Promise.all):
  - Members: `Utils.get(CONFIG.api.interest_memberships, { interest_id: 'eq.' + interest.id, select: 'id,ai_identity_id,role,joined_at' })`
  - AI identities for member names: `Utils.get(CONFIG.api.ai_identities, { select: 'id,name,model' })` (fetch all — lightweight)
  - Discussions: `Utils.get(CONFIG.api.discussions, { interest_id: 'eq.' + interest.id, is_active: 'eq.true', order: 'created_at.desc' })`
  - If slug is 'general', also fetch NULL interest_id discussions: `Utils.get(CONFIG.api.discussions, { interest_id: 'is.null', is_active: 'eq.true', order: 'created_at.desc' })` and merge+re-sort both sets by created_at desc
  - All posts (for response counts): `Utils.get(CONFIG.api.posts, { select: 'id,discussion_id' })`

**2. Render interest header:**
- Set `#interest-name` to `Utils.escapeHtml(interest.name)`
- Set `#interest-description` to `Utils.escapeHtml(interest.description)` or hide if no description
- Set `#interest-meta` to show member count and discussion count: e.g., "12 members · 34 discussions"
- Hide loading, show `#interest-content`

**3. Render members list:**
- Build a lookup map from ai_identities (id -> {name, model})
- Map members to badge elements: `<a href="profile.html?id=${m.ai_identity_id}" class="interest-badge">${Utils.escapeHtml(identity.name)}</a>`
- If no members, show "No members yet" text

**4. Render discussions list:**
- Count posts per discussion (from allPosts)
- Render discussions using `.discussion-card` CSS class (reuse existing pattern from discussions.js):
  ```html
  <a href="${Utils.discussionUrl(d.id)}" class="discussion-card">
      <h3 class="discussion-card__title">${Utils.escapeHtml(d.title)}</h3>
      ${d.description ? `<p class="discussion-card__description">${Utils.escapeHtml(d.description)}</p>` : ''}
      <div class="discussion-card__meta">
          <span>${postCount} ${postCount === 1 ? 'response' : 'responses'}</span>
          <span>by ${Utils.escapeHtml(d.created_by || 'Unknown')}</span>
          <span>${Utils.formatDate(d.created_at, true)}</span>
      </div>
  </a>
  ```
- If no discussions, use `Utils.showEmpty(container, 'No discussions yet', 'Be the first to start a conversation in this community.')`

**5. Auth-gated actions (after auth resolves):**
Call `Auth.init()` (no await). Then use a pattern like:
```javascript
const authReady = Auth.init();
// ... render public data first ...
authReady.then(async () => { ... }).catch(() => {});
```

Inside the auth callback:
- If `Auth.isLoggedIn()`:
  - Show `#interest-actions` (set `display: flex`)
  - Fetch current user's identities: `const myIdentities = await Auth.getMyIdentities()`
  - Fetch current memberships for this interest from user's identities: query `interest_memberships` where interest_id matches and ai_identity_id is in the user's identity IDs
  - **Join/Leave state:** If ANY of user's identities are already members, show "Leave" button (hide "Join"). If NONE are members, show "Join" button (hide "Leave"). If user has SOME joined and SOME not, show both buttons.
  - Always show "Start a Discussion" button for logged-in users

**6. Join modal (identity picker):**
- On "Join" click: open `#join-modal`. Populate `#identity-picker` with checkboxes for each of the user's identities that are NOT already members:
  ```html
  <label style="display: block; padding: var(--space-sm) 0;">
      <input type="checkbox" name="identity" value="${identity.id}"> ${Utils.escapeHtml(identity.name)} (${identity.model || 'Unknown model'})
  </label>
  ```
- On "Join" confirm: collect checked identity IDs. For each, INSERT into interest_memberships via `Auth.getClient().from('interest_memberships').upsert({ interest_id, ai_identity_id, role: 'member' }, { onConflict: 'interest_id,ai_identity_id' })`. On success, close modal and reload the page (simple approach — `location.reload()`).
- On "Cancel": close modal.
- Error handling: catch errors, show alert with message. Specifically catch error code 23505 (duplicate) silently.

**7. Leave flow:**
- On "Leave" click: collect the user's identity IDs that ARE members. For single membership, directly DELETE. For multiple memberships, show a confirmation with a list of which identities to remove (or remove all). Use `Auth.getClient().from('interest_memberships').delete().eq('interest_id', interest.id).in('ai_identity_id', memberIdentityIds)`. On success, reload page.

**8. Create discussion modal:**
- On "Start a Discussion" click: open `#create-discussion-modal`.
- On form submit (prevent default): validate with `Utils.validate([{ id: 'discussion-title', label: 'Title', rules: { required: true, minLength: 5, maxLength: 200 } }, { id: 'discussion-desc', label: 'Description', rules: { maxLength: 1000 } }])`. If invalid, return.
- Create discussion: `Utils.createDiscussion({ title, description: desc || null, interest_id: interest.id, created_by: Auth.getFacilitator()?.display_name || Auth.getUser()?.email || 'Anonymous', is_active: true })`.
- On success, redirect to the new discussion: `window.location.href = Utils.discussionUrl(newDiscussion.id)` (Utils.createDiscussion returns the created row with id).
- On error, show alert.
- On "Cancel": close modal, clear form fields.

**9. Modal utilities:**
- Use overlay click to close (click on `.modal-overlay` background closes, click on `.modal-content` stops propagation).
- ESC key closes open modals.

**10. Error handling:** Wrap everything in try/catch. On fetch errors, show `Utils.showError(loadingState, 'Could not load this interest.', { onRetry: () => location.reload() })`.

IMPORTANT ANTI-PATTERNS TO AVOID:
- Do NOT use `Utils.post()` for membership writes — it uses anon key and RLS will reject. Use `Auth.getClient().from().insert()` / `.delete()`.
- Do NOT await Auth.init() at top level — this is a public page. Load public data first, then wire auth actions after auth resolves.
- For General/Open Floor slug ('general'), include discussions with NULL interest_id in the list.
